# StackUp

Веб-компаньон для домашнего покера: учёт стеков фишек вместо физических фишек.
Без карт, без онлайн-игры на деньги — организатор создаёт стол, игроки заходят по
QR-коду и вводят только имя, организатор вручную фиксирует rebuy/add-on и
корректировки стека, в конце — кнопка «Завершить игру» показывает итоги.

## 1. Настройка Supabase

1. Создайте проект на [supabase.com](https://supabase.com).
2. Откройте **SQL Editor** → по очереди выполните `0001_init.sql`, `0002_player_status.sql`,
   `0003_poker_table.sql`, `0004_avatars.sql`, `0005_avatar_set_update.sql`,
   `0006_betting.sql`, `0007_award_pot.sql`, `0008_win_banner.sql`,
   `0009_side_pots.sql`, `0010_leave_table.sql`, `0011_hand_history.sql`,
   `0012_finish_game_guard.sql`, `0013_turn_timer.sql`, `0014_betting_fixes.sql`,
   а затем `0015_unique_names.sql`
   (каждый — отдельным запросом, строго по порядку).
   Это создаст таблицы, RLS-политики и все нужные функции, а также добавит
   `rooms` и `players` в Realtime-публикацию.
3. В **Project Settings → API** скопируйте `Project URL` и `anon public` ключ.

## 2. Настройка проекта

```bash
cp .env.example .env
# впишите в .env свои VITE_SUPABASE_URL и VITE_SUPABASE_ANON_KEY

npm install
npm run dev
```

Приложение откроется на `http://localhost:5173`.

## 3. Деплой

Это статичный SPA — после `npm run build` папку `dist` можно отдать любым
статическим хостингом (Vercel, Netlify, Cloudflare Pages и т.д.), сервер не
нужен: фронтенд обращается к Supabase напрямую через REST/Realtime. Не
забудьте задать те же переменные окружения (`VITE_SUPABASE_URL`,
`VITE_SUPABASE_ANON_KEY`) в настройках хостинга.

QR-код в панели организатора кодирует ссылку вида `<ваш-домен>/join/<id>` —
поэтому деплой нужно сделать один раз, до того как звать игроков.

## Как это работает

- **Организатор**: Home → «Создать стол» → получает приватный `host_token`
  (хранится только в localStorage этого браузера) → попадает в `/host/:roomId`
  с QR-кодом и списком игроков.
- **Игрок**: сканирует QR → вводит имя на `/join/:roomId` → видит свой стек на
  `/play/:roomId`, который обновляется в реальном времени.
- **Завершение**: организатор жмёт «Завершить игру» → все участники
  автоматически переходят на `/results/:roomId`.
- **Игрок ушёл раньше времени**: кнопка «✕» на карточке игрока в панели
  организатора фиксирует его текущий стек как финальный результат и убирает
  из списка «за столом» — баланс стола (`sum(result) = 0`) при этом
  продолжает сходиться, а в итогах он всё равно появится.
- **История операций**: сворачиваемый список под столом игроков — все
  входы, rebuy/add-on и ручные корректировки с именем игрока и временем.
- **Светлая/тёмная тема**: переключатель ☀️/🌙 в правом верхнем углу,
  выбор запоминается в localStorage; по умолчанию ориентируется на
  системную тему устройства.

### Этап 1 — настоящий покерный стол (новое)

- **Хост тоже может играть**: на экране лобби есть форма «Сесть за стол» —
  хост получает свой стек и место за столом, не теряя прав администратора.
- **Лобби перед игрой**: после создания стола игра не начинается сразу —
  организатор настраивает блайнды, ждёт игроков, и только после нажатия
  «Начать игру» комната переходит в активную фазу.
- **Визуальный стол**: овальный стол с местами по кругу, баттон дилера,
  значки SB/BB, контейнер банка по центру (сам банк подключим на
  следующем этапе вместе со ставками).
- **Ротация дилера и блайндов**: кнопка «Следующая раздача» двигает
  баттон дилера по кругу и увеличивает номер раздачи; блайнды можно
  менять в любой момент через кнопку «Блайнды».
- **Важное ограничение этого этапа**: реальных ставок, банка и
  автоматического списания блайндов ещё нет — фишками всё ещё управляет
  организатор вручную («Стек» / «Rebuy»), как и раньше. Это будет
  следующим этапом.

### Этап 2а — реальные ставки (новое)

- **Ходы игроков**: Чек / Колл / Бет / Рейз / Пас / Олл-ин — каждый
  игрок ходит сам со своего телефона в свой ход (подсвечивается на столе
  зелёным кольцом), стек и банк считаются автоматически.
- **Блайнды теперь списываются сами**: при нажатии «Начать игру» и
  «Следующая раздача» блайнды сразу списываются с нужных игроков, и
  открывается круг торгов — отдельной кнопки «раздать» нет.
- **Героз-ап (ровно 2 игрока)**: используется настоящее покерное
  правило — дилер ставит малый блайнд и ходит первым. При 3+ игроках
  используется упрощённая схема «по кругу после дилера» (как и раньше
  для визуальных значков SB/BB).
- **Если все, кроме одного, сбросили карты** — банк автоматически
  уходит этому игроку, раздача завершается сама.
- **Сайд-поты теперь считаются корректно** — см. «Этап 2г» ниже.

### Выбор аватара (новое)

При входе по QR-коду игрок сначала вводит имя, затем выбирает один из
12 готовых аватаров — картинки взяты из макета, который прислал заказчик
(сгенерированы им через ChatGPT, использование разрешено), вырезаны и
оптимизированы под интерфейс (`public/avatars/*.png`, 160×160). Если
игрок просто нажимает «Сесть за стол», не трогая сетку, ему всё равно
достаётся случайный аватар по умолчанию (сервер подстрахует это и на
уровне RPC-функции). Аватар отображается на месте игрока за столом —
картинка вписана внутрь фишки, фирменный рубчатый ободок остаётся
поверх. Хост получает тот же выбор в форме «Сесть за стол» в лобби.
Места за столом по-прежнему назначаются автоматически по порядку входа —
выбор конкретного места вручную не делали, чтобы не раздувать эту фичу;
если он нужен, скажите — добавим отдельно.

### Обновлённый визуал стола (новое)

Цветовая палитра интерфейса (не стола, а именно фона/карточек/текста)
пересобрана под более современный «покерный» вид: нейтральный
тёмно-синий фон вместо зелёного оттенка, янтарный акцент для ключевых
кнопок и баттона дилера, отдельный фиксированный зелёный токен для
самого сукна стола. Места за столом теперь оформлены как карточки с
рамкой, а не просто текст под фишкой — это заодно исправляет баг с
наложением текста «Раздача №N» на верхнее место за столом (карточки
теперь гарантированно помещаются в границы стола при любом числе
игроков). Светлая тема не трогалась.

### Этап 2б — выбор победителя и расчёт переводов (новое)

- **Экран «Завершить раздачу»**: когда торги закончены (и остался не один
  игрок — иначе банк, как и раньше, уходит ему автоматически), у
  организатора появляется кнопка «Завершить раздачу». Она открывает
  список ещё не сбросивших карты игроков с чекбоксами — можно выбрать
  одного победителя или нескольких сразу. Приложение **не определяет
  победителя само** — это делают игроки по физическим картам, организатор
  только отмечает результат.
- **Сплит банка**: если выбрано несколько победителей, банк делится
  поровну; если не делится без остатка, «лишние» фишки достаются по
  одной игрокам по порядку мест за столом (детерминированно, чтобы ничего
  не терялось).
- **Итоги игры → «Кто кому платит»**: на экране итогов под таблицей
  результатов появился блок минимальных переводов — кто конкретно кому
  переводит и сколько, посчитано так, чтобы число переводов было
  минимальным (тот же принцип, что в Splitwise: должник с самым большим
  долгом гасит его получателю с самым большим профитом). Суммы — в тех же
  фишках, что и весь остальной учёт; если при создании стола указана
  валюта, она просто подписывается рядом как единица измерения (1 фишка =
  1 единица валюты) — это самый простой и понятный вариант, поскольку
  rebuy/add-on вводятся в фишках и могут не быть кратны сумме входа,
  значит фишки не всегда честно конвертируются в деньги. Если нужен
  другой принцип (например — отдельно ввести сумму входа в деньгах),
  скажите.

### Этап 2в — визуал под присланный макет + анимация (новое)

- **Стопки фишек вместо плоского текста**: банк в центре и ставка
  каждого игрока теперь отрисованы как маленькая иконка стопки фишек
  (CSS, без картинок), а не просто число.
- **«Перелёт» фишек к банку**: когда игрок ставит/коллирует/рейзит,
  от его места к центру стола летит фишка — стол не статичный, видно,
  что что-то произошло, причём у всех за столом одновременно (через
  обычное обновление состояния, без отдельных «событий анимации»).
- **Баннер «Х забирает банк +N»**: показывается на пару секунд всем
  участникам сразу после того, как раздача распределена (и при ручном
  выборе через «Завершить раздачу», и при автоматической победе по
  фолду) — данные о последнем победителе хранятся прямо в комнате
  (`last_hand_winner_ids`/`last_hand_pot`), поэтому баннер синхронный у
  всех, а не только у того, кто нажал кнопку. От центра стола к месту
  каждого победителя при этом летит фишка (тот же механизм, что и для
  обычных ставок, просто в обратную сторону), а его карточка на пару
  секунд подсвечивается золотым — то самое «АНИМАЦИЯ ВЫИГРЫША» из
  макета.
- **«Ходит: Имя»** — отдельная строка над столом в свой ход показывает,
  кто сейчас действует (как «NEXT TO ACT» в макете), плюс зелёное
  кольцо на месте этого игрока, как и раньше.
- **Панель действий — 4 кнопки**: Пас / Чек-Колл / Рейз / Олл-ин — как
  на макете, отдельной кнопкой, без переключения. Под ней —
  степпер суммы и три круглые «фишки»-пресета (MIN / 1/2 банка / БАНК).
  Сами суммы пресетов остались прежними (доля банка, а не фиксированные
  номиналы 1/5/25/100/500 как в макете) — это специально, чтобы кнопки
  оставались полезными при любых блайндах/стеках, а не только при тех
  конкретных цифрах, что были на скетче. Если важнее точное визуальное
  соответствие, чем универсальность — скажите, поменяю на фиксированные.
- **Дилер/SB/BB**: у значков SB/BB теперь подписан размер блайнда
  (например «SB 10»), не только буквы.

**Чего из макета здесь всё ещё нет** (сознательно, чтобы не раздувать и
этот шаг до бесконечности): шапка стола «Room: ... / Blinds: ... /
LEAVE TABLE» с меню-гамбургером — сейчас шапка в собственном стиле
приложения, без этих элементов (кнопка «Покинуть стол» есть, но в
шапке игрока, не организатора — см. «Этап 2д»). Настоящий таймер хода
(на макете «8s») и анимация перелёта фишек от банка к победителю по
координатам — реализованы отдельно, см. ниже.

### Этап 2г — сайд-поты (новое)

Раньше при олл-ине за меньший стек, если другие игроки продолжали
ставить дальше, всё уходило в один общий банк — это была явно
обозначенная неточность. Теперь при завершении раздачи банк сам
режется на слои по стандартному покерному алгоритму: основной банк
(в котором участвуют все, включая того, кто пошёл в олл-ин) и один или
несколько побочных (только для тех, кто продолжал ставить дальше).
Экран «Завершить раздачу» показывает каждый банк отдельной карточкой со
своей суммой и своим списком тех, кто имеет на него право — победителя
выбираете отдельно для каждого банка. Если сайд-потов в конкретной
раздаче не возникло (обычный случай), экран выглядит как раньше — один
банк, один список. Разбивка считается заново на сервере в момент
начисления (не доверяя тому, что мог насчитать браузер), так что
подменить суммы из интерфейса нельзя.

### Этап 2д — «Покинуть стол» и история раздач (новое)

- **Игрок может выйти сам**, без участия организатора — кнопка
  «Покинуть стол» в шапке экрана `/play/:roomId` (как LEAVE TABLE в
  макете). Зафиксированный стек засчитывается в итоги как обычно при
  уходе. **Ограничение**: уйти можно только между раздачами, не
  посреди торгов — иначе фишки, уже внесённые в текущий банк, могли бы
  «потеряться» из расчёта сайд-потов. Если раздача идёт, кнопка отдаст
  понятную ошибку с просьбой дождаться её завершения.
- **История раздач** — сворачиваемый список под историей операций в
  панели организатора: номер раздачи, кто выиграл (один или несколько,
  при сплите/сайд-потах), сумма, время. Хранится отдельно от
  «последнего результата» (того, что показывает баннер) — не
  перезаписывается, накапливается за всю игру.

### Исправление — «Завершить игру» больше не блокируется и не ошибается

Раньше была найдена реальная дыра: если завершить игру прямо посреди
раздачи, фишки из банка пропадали из итогового баланса без объяснения —
и первое исправление было «блокировать кнопку, пока банк не
распределён». По дополнительному фидбеку это оказалось неправильным
поведением: организатор должен мочь завершить игру в любой момент, без
ошибок. Теперь так и есть — `finish_game` никогда не падает с ошибкой:
если раздача не доведена до конца, все ставки этого круга торгов
(банк) возвращаются игрокам обратно в стек (как будто раздачи и не
было — победитель для незаконченной раздачи не объявляется, это было
бы произвольным решением), и только потом сессия закрывается. Кнопка в
интерфейсе теперь всегда активна; если раздача не завершена, под ней
просто поясняющая строка о том, что банк вернётся в стек.

### Исправление — реальный баг в логике торгов (Call не передавал ход)

Самое серьёзное найденное по фидбеку: круг торгов закрывался
преждевременно в самом частом случае — когда никто не рейзит, а просто
коллируют блайнд по кругу. Раньше закрытие круга определялось
сравнением «следующий по очереди == тот, кто ставил последним», а
последним «ставившим» считался big blind сразу после постановки
блайнда (это технический артефакт, не настоящий рейз) — из-за этого
действие, дойдя обратно до BB, закрывало круг, даже не дав BB сходить
(ни чек, ни рейз) — то есть последнему игроку в очереди ход просто не
передавался. Баг проявлялся и один на один, и за полным столом.

Исправлено переходом на честный учёт: у каждого игрока теперь есть флаг
«ходил ли он в этом круге торгов после последней ставки/рейза». Круг
закрывается только тогда, когда не осталось ни одного живого игрока, который
либо ещё не ходил, либо ещё не уровнял текущую ставку. Любая ставка/рейз
заново сбрасывает этот флаг всем остальным живым игрокам — им нужно
отреагировать на новую сумму, даже если уже ходили раньше в этом же
круге. Проверено вручную на нескольких раскладах: обычный круг без
рейзов (один на один и за полным столом), повторный рейз, олл-ин за
меньший стек с сайд-потом — везде ход теперь передаётся туда, куда
должен.

### Стопки фишек на столе (новое)

Перед каждым игроком на столе теперь лежит своя стопка фишек —
количество дисков в ней пропорционально его текущему стеку относительно
самого большого стека за столом (а не абсолютным числам, чтобы работало
при любых блайндах/стартовых стеках). При ставке несколько фишек
реально срываются с этой стопки и едут в банк (раньше летел один
кружок — теперь горсть из 2-3 фишек, выглядит живее). У игрока с нулевым
стеком стопки нет вообще, у сфолдившего — не показывается, пока он не
зайдёт в следующую раздачу.

### Шапка стола и уникальность имён (новое)

У организатора блайнды в шапке теперь не просто текст, а кнопка — тап по
ним сразу открывает редактирование, без отдельной строки кнопок ниже
(она всё ещё там же, это просто более быстрый доступ к тому же самому).
Отдельно — раньше два игрока могли присоединиться к одному столу с
одинаковым (или отличающимся только регистром) именем; на экране выбора
победителя, в расчёте переводов, в истории раздач это было бы
неразличимо. Теперь имя проверяется на совпадение с уже сидящими за
столом, и попытка занять занятое имя получает понятную ошибку с
просьбой выбрать другое.

### Расчёт в реальных деньгах (новое)

Раньше переводы между игроками считались в фишках 1:1 с просто
подписанной валютой рядом — по фидбеку это было неправильно. Теперь,
если при создании стола указана сумма бай-ина в деньгах
(`buy_in_amount`), курс пересчёта считается как
`деньги_за_бай-ин / стартовый_стек_в_фишках`, и применяется одинаково и
к вложенному (бай-ины + ребаи), и к финальному стеку каждого игрока —
ровно по формуле и примеру из фидбека (1000 фишек = 500 грн → 3000
фишек в конце = 1500 грн при вложенных 1000 грн = результат +500 грн).
Переводы между игроками («кто кому платит») теперь считаются и
показываются в деньгах, если курс известен. На экране итогов под
результатом в фишках (фишки остаются — удобно сверить с реальной
стопкой на столе) появляется такой пересчёт в скобках. Если сумму
бай-ина при создании стола не указали — всё считается в фишках, как
раньше (курса для пересчёта просто нет).

### Таймер хода (новое)

На размышление даётся 25 секунд — отсчёт виден всем за столом (не
только тому, чей ход) рядом с «Ходит: Имя», а у самого активного игрока
ещё и шкала над кнопками действий, краснеющая на последних секундах.
Если время вышло, система делает действие за игрока сама: Чек, если
нечего уравнивать, иначе Фолд — то же самое, что произошло бы при ручном
бездействии за столом в реальной игре.

Технически отдельного сервера с крон-джобами нет (Supabase — это просто
функции по запросу), поэтому таймаут «ловит» любой из открытых экранов:
и панель организатора, и каждый телефон игрока тихо проверяют раз в
пару секунд, не истекло ли время. Это специально избыточно — если у
того, чей ход, разрядился телефон или закрылась вкладка, кто-то другой
всё равно это заметит и подтолкнёт игру дальше. На сервере это
безопасно — обработка таймаута «забирается» атомарно, так что даже если
несколько экранов одновременно это заметят, реальное действие
выполнится только один раз.

### PWA — установка на главный экран (новое)

Приложение можно поставить на телефон как обычное (manifest.json,
иконки, простой service worker). Service worker кэширует только саму
оболочку приложения (HTML/JS/CSS со своего домена) по стратегии «сеть,
потом кэш» — после каждого деплоя всегда подтягивается свежая версия,
кэш используется только как подстраховка при обрыве связи. Сама игра
всё равно требует сети (нужна синхронизация между телефонами через
Supabase Realtime) — офлайн-режима для самой игры нет и не планировался,
PWA здесь даёт только быстрый значок на экране и более похожий на
нативное приложение вид (без адресной строки браузера).

Вся бизнес-логика выполняется через Postgres RPC-функции
(`create_room`, `join_room`, `host_join_room`, `start_game`, `next_hand`, `player_action`, `check_timeout`, `award_pots`,
`leave_table`, `adjust_stack`, `add_rebuy`, `remove_player`, `finish_game`) —
прямые INSERT/UPDATE из браузера запрещены через RLS. `host_token` хранится в
отдельной таблице `room_hosts`, а персональный токен каждого игрока — в
`player_tokens`; обе таблицы без единой select-политики, поэтому токены не
протекают ни через обычные запросы, ни через Realtime-подписки.
